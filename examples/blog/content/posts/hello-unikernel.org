** What is a unikernel?

A /unikernel/ is a specialised, single-address-space machine image constructed by using library operating systems. In plain English: your application *is* the operating system. No Linux kernel, no userspace, no shell. Just your code and the libraries it needs, compiled into a single binary.

** Why MirageOS?

MirageOS lets you write unikernels in OCaml. The entire network stack, filesystem, and HTTP server are OCaml libraries. The result is a tiny, fast, secure appliance that boots in milliseconds.

Some benefits:

- *Small attack surface* --- no shell, no unnecessary services
- *Fast boot* --- typically under 100ms
- *Type safety* --- OCaml catches entire classes of bugs at compile time
- *Reproducible* --- Nix builds the whole thing deterministically

** A code example

Here's the core of the HTTP dispatcher:

#+BEGIN_SRC ocaml
let rec dispatcher fs uri =
  match Uri.path uri with
  | "" | "/" -> dispatcher fs (Uri.with_path uri "/index.html")
  | path ->
      try_serve fs path >>= function
      | Some resp -> resp
      | None ->
          let index_path = path ^ "/index.html" in
          try_serve fs index_path >>= function
          | Some resp -> resp
          | None -> Http.respond_not_found ()
#+END_SRC

Simple pattern matching. Try the exact path, fall back to =path/index.html= for clean URLs, return 404 otherwise.
